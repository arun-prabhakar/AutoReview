Features

Everything you need for
automated code review

From on-demand commit analysis to scheduled PR reviews — AutoReview covers the entire review lifecycle.

Manual & PR Reviews

Review individual commits or full pull requests on demand from the web UI.

Automatic Reviews

Configure polling to detect new commits and PR updates. Reviews run automatically in the background.

Structured Findings

Findings classified as Must Fix, Should Fix Soon, or Ignore — with file, line, category, explanation, and suggested fix.

Multi-Provider LLM

Supports OpenAI, Anthropic, Google Gemini, AWS Bedrock, and any OpenAI-compatible API. Per-repository provider and model configuration.

Multi-Pass Review

Optionally run specialized passes for security, performance, and maintainability, then deduplicate findings.

Review History

Re-run reviews while preserving earlier results in a linked history chain. Full audit trail.

Sharing

Create public share links with optional expiry for external stakeholders.

Bitbucket Integration

Post PR comments and inline findings back to Bitbucket. Automatic or manual.

Email Notifications

Generate review email drafts and send via SMTP. Configurable per repository.

Cost Analytics

Track token usage, estimated costs by model, and per-review spending with detailed dashboards.

Prompt Templates

Customizable prompt editor with variable placeholders, AI enhancement, and strictness-level tabs.

Enterprise Security

AES-256-GCM encryption for all secrets. JWT auth with HTTP-only cookies. Role-based access.

Quick Start

Up and running in minutes

0 Prerequisites

  • › Node.js 20+
  • › PostgreSQL database
  • › Bitbucket Cloud app password
  • › An LLM provider API key (OpenAI, Anthropic, Gemini, or AWS Bedrock credentials)

1 Server Setup

cd server cp .env.example .env # Edit .env with DATABASE_URL, ENCRYPTION_KEY, JWT_SECRET npm install npm run db:init npm run dev

The API runs on http://localhost:3001

2 Client Setup

cd client npm install npm run dev

The client runs on http://localhost:5173 and proxies /api to the server.

Deployment

Docker Deployment

Single-command deployment with Docker Compose. The built client and API are served together.

export ENCRYPTION_KEY=your_64_character_hex_key export JWT_SECRET=your_jwt_secret export DATABASE_URL=postgresql://user:password@host:5432/database docker compose up -d --build

The Docker setup uses a multi-stage build. The client is built during the builder stage and served as static files by the Express server. Port 3000 maps to internal port 3001.

How It Works

Review Lifecycle

From diff to delivered findings — every step is automated, auditable, and configurable.

Fetch Diff

AutoReview fetches the commit or PR diff from Bitbucket Cloud using configured credentials.

Build Prompt

The review prompt is assembled from the configured template, repository strictness, and optional .autoreview.md project context.

LLM Analysis

The prompt is sent to the configured LLM provider — OpenAI, Anthropic, Gemini, AWS Bedrock, or any OpenAI-compatible endpoint.

Parse & Store

Structured JSON findings are parsed, validated, deduplicated, and stored in PostgreSQL with full audit history.

Generate Overview

A compact, validated AI overview sentence is generated for dashboards and email drafts.

Notify & Deliver

Findings are delivered via email, Bitbucket PR comments, shareable links, and in-app notifications.
